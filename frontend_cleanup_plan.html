2. Identified Issues

CSS Issues

🔴 Issue 1: Monolithic CSS File (Priority: HIGH)

Problem: All 785 lines of CSS are in a single file with minimal organization.

  • Difficult to maintain and navigate
  • No clear separation of concerns (layout, components, utilities)
  • Hard to identify unused styles
  • Limited comments/sections (only 8 section comments)

🔴 Issue 2: CSS Class Naming Inconsistency (Priority: HIGH)

Problem: Multiple naming conventions used throughout:

  • BEM-like: card-base, card-header, card-table-header
  • Utility-based: flex, hidden
  • Grid-specific: purchase-horizontal-grid, budget-horizontal-grid-yearly
  • Context-specific: rollover-edit, menu-toggle

No clear methodology makes classes hard to predict and reuse.

🟡 Issue 3: Excessive Grid Layout Classes (Priority: MEDIUM)

Problem: 15+ specific grid classes for different layouts:

.summary-horizontal-grid-monthly .purchase-horizontal-grid .purchase-horizontal-grid-categories .purchase-horizontal-grid-delete .income-horizontal-grid-monthly .recurring-purchase-grid-full .recurring-purchase-delete-grid .recurring-add-grid-full .budget-horizontal-grid-yearly .budget-horizontal-grid-monthly .budget-horizontal-grid-month ...and more

These could be consolidated using utility classes or a more flexible grid system.

🟡 Issue 4: Commented-out Code (Priority: LOW)

Problem: Several instances of commented-out CSS:

  • Lines 66, 143-146, 352-355, 502-506, 376, 779-786
  • Makes file harder to read
  • Unclear if code should be removed or kept for reference

JavaScript Issues

🔴 Issue 5: No Separate JavaScript Files (Priority: HIGH)

Problem: All custom JavaScript is inline in HTML templates:

  • _base.html: 54 lines of modal and menu logic
  • yearly_budget_detail.html: 56 lines for rollover editing and YTD selector
  • purchase_create.html: 56 lines for dynamic form management

Consequences:

  • JavaScript not cached by browser (loaded with every page)
  • Cannot be minified or bundled
  • Harder to test in isolation
  • Code reuse impossible (e.g., modal logic duplicated conceptually)
  • No IDE support for JavaScript in template files

🟡 Issue 6: Duplicate CSRF Token Handling (Priority: MEDIUM)

Problem: getCookie() function defined inline in yearly_budget_detail.html, but CSRF token also handled via HTMX headers in base template.

Should use a consistent approach across all JavaScript.

🟡 Issue 7: Console.log Statements in Production Code (Priority: LOW)

Problem: Debug console.log statements in purchase_create.html (lines 65, 67, 75)

Should be removed or wrapped in development checks.

HTML/Template Issues

🔴 Issue 8: Repetitive HTML Patterns (Priority: HIGH)

Problem: Same HTML structures repeated across multiple templates:

  • Edit/Delete Button Pattern: Found 10+ times
    <div class="edit-links-container"> <button class="button-blank" hx-get='...' hx-target="#modal-content"> <img src="edit-pencil.svg" width="17px"> </button> <button class="button-blank" hx-get='...' hx-target="#modal-content"> <img src="trash.svg" width="17px"> </button> </div>
  • Card Table Headers: Found 25+ times
  • Modal Structure: Similar structure in 11 modal templates
  • Action Buttons: "Add Purchase", "Add Income" repeated across pages

🟡 Issue 9: No Template Partials/Includes (Priority: MEDIUM)

Problem: Not leveraging Django's {% include %} tag for reusable components.

Common patterns like action buttons, table rows, modal wrappers could be extracted.

🟡 Issue 10: Inline Styles (Priority: LOW)

Problem: One instance found: style="flex-grow:1;" in yearly_budget_detail.html

Should be moved to CSS classes.

🟢 Issue 11: Icon Sizing Inconsistency (Priority: LOW)

Problem: SVG icon widths hardcoded as "17px" in HTML rather than CSS.

Should be standardized through CSS classes.

// static/js/core/modal.js export class Modal { constructor() { this.modal = document.querySelector("#modal"); this.modalContent = document.querySelector("#modal-content"); this.overlay = document.querySelector(".overlay"); this.init(); } init() { // HTMX integration htmx.on("htmx:afterSwap", (e) => { if (e.detail.target.id === "modal-content") { this.show(); this.focusFirstElement(); } }); // Close on overlay click this.overlay.addEventListener("click", (event) => { if (!event.target.closest(".modal")) { this.close(); } }); // Close on Escape key window.addEventListener("keydown", (event) => { if (event.key === "Escape") { this.close(); } }); } show() { this.modal.classList.remove("hidden"); this.overlay.classList.remove("hidden"); this.modal.scrollTo(0, 0); } close() { this.modal.classList.add("hidden"); this.overlay.classList.add("hidden"); } focusFirstElement() { const firstInteractive = this.modalContent.querySelector( 'input:not([type="hidden"]):not([disabled]), ' + 'select:not([disabled]), textarea:not([disabled]), ' + 'button:not([disabled]), a[href], [tabindex]:not([tabindex="-1"])' ); firstInteractive?.focus(); } }

Step 1.2: Create core/utils.js

Centralize utility functions:

// static/js/core/utils.js /** * Get CSRF token from cookies */ export function getCsrfToken() { const name = 'csrftoken'; if (document.cookie && document.cookie !== '') { const cookies = document.cookie.split(';'); for (let cookie of cookies) { cookie = cookie.trim(); if (cookie.startsWith(name + '=')) { return decodeURIComponent(cookie.substring(name.length + 1)); } } } return null; } /** * Format currency values */ export function formatCurrency(amount) { return new Intl.NumberFormat('en-US', { style: 'currency', currency: 'USD' }).format(amount); }

Phase 2: Extract Feature JavaScript

Step 2.1: Create features/rollover-editor.js

Extract rollover editing from yearly_budget_detail.html (lines 312-365):

// static/js/features/rollover-editor.js import { getCsrfToken } from '../core/utils.js'; export class RolloverEditor { constructor(year) { this.year = year; this.csrfToken = getCsrfToken(); this.inputs = document.querySelectorAll(".rollover-edit"); this.init(); } init() { this.inputs.forEach((input) => { input.addEventListener('keydown', (e) => this.handleKeydown(e)); input.addEventListener('blur', (e) => this.save(e)); }); } handleKeydown(event) { if (event.key === "Enter" || event.type === 'blur') { this.save(event); } } async save(event) { event.preventDefault(); const amount = event.currentTarget.value; const category = event.currentTarget.dataset.category; try { const response = await fetch('/budgets/rollover/update/', { method: 'POST', credentials: 'same-origin', headers: { 'Accept': 'application/json', 'X-Requested-With': 'XMLHttpRequest', 'X-CSRFToken': this.csrfToken, }, body: JSON.stringify({ amount, category, year: this.year }) }); if (!response.ok) { console.error('Failed to save rollover'); } } catch (error) { console.error('Error saving rollover:', error); } } }

Step 2.2: Create features/purchase-form.js

Extract dynamic form management from purchase_create.html (lines 24-80):

// static/js/features/purchase-form.js export class PurchaseFormManager { constructor() { this.container = document.querySelector("#purchase-form-container"); this.purchaseForms = document.querySelectorAll(".purchase-form"); this.addButton = document.querySelector("#add-form"); this.submitButton = document.querySelector("#submit-button"); this.totalFormsInput = document.querySelector("#id_form-TOTAL_FORMS"); this.totalDisplay = document.querySelector('#total'); this.formNum = this.purchaseForms.length - 1; this.init(); } init() { this.addButton.addEventListener('click', (e) => this.addForm(e)); const amountInputs = document.querySelectorAll("[id$='-amount']"); amountInputs.forEach(input => { input.addEventListener('change', (e) => this.calculateTotal(e)); }); } addForm(e) { e.preventDefault(); const newForm = this.purchaseForms[0].cloneNode(true); const formRegex = /form-(\d){1}-/g; this.formNum++; newForm.innerHTML = newForm.innerHTML.replace( formRegex, `form-${this.formNum}-` ); this.container.insertBefore(newForm, this.submitButton); this.totalFormsInput.setAttribute('value', `${this.formNum + 1}`); const newAmountInput = document.querySelector( `#id_form-${this.formNum}-amount` ); newAmountInput.addEventListener('change', (e) => this.calculateTotal(e)); document.querySelector(`#id_form-${this.formNum}-item`).focus(); } calculateTotal(e) { e.preventDefault(); const amounts = document.querySelectorAll("[id$='-amount']"); const total = [...amounts].reduce((sum, input) => { const value = parseFloat(input.value) || 0; return sum + value; }, 0); this.totalDisplay.textContent = `Total: $${total.toFixed(2)}`; } }

Note: Removed console.log statements and improved error handling.

// static/js/main.js import { Modal } from './core/modal.js'; import { MobileNav } from './features/navigation.js'; import { RolloverEditor } from './features/rollover-editor.js'; import { PurchaseFormManager } from './features/purchase-form.js'; document.addEventListener('DOMContentLoaded', () => { // Initialize core features (on every page) new Modal(); new MobileNav(); // Initialize page-specific features if (document.querySelector('.rollover-edit')) { const year = document.querySelector('[data-year]')?.dataset.year; new RolloverEditor(year); } if (document.querySelector('#purchase-form-container')) { new PurchaseFormManager(); } // Initialize YTD month selector (if present) const monthSelect = document.querySelector('.month-select'); if (monthSelect) { initYtdSelector(monthSelect); } }); function initYtdSelector(select) { const urlParams = new URLSearchParams(window.location.search); const ytd = urlParams.get('ytd'); if (ytd) { select.selectedIndex = parseInt(ytd) - 1; } select.addEventListener('change', function() { const currentPath = window.location.pathname; window.location.href = `${currentPath}?ytd=${this.value}`; }); }

6. HTML/Template Refactoring Plan

Phase 1: Create Reusable Template Partials

Step 1.1: Create _partials/edit-delete-buttons.html

{# templates/_partials/edit-delete-buttons.html #} {# Usage: {% include '_partials/edit-delete-buttons.html' with edit_url='...' delete_url='...' %} #} {% load static %} <div class="edit-links-container"> <button class="button-blank" hx-get="{{ edit_url }}" hx-target="#modal-content" aria-label="Edit"> <img class="icon icon--sm" src="{% static 'images/icons/edit.svg' %}" alt="Edit"> </button> <button class="button-blank" hx-get="{{ delete_url }}" hx-target="#modal-content" aria-label="Delete"> <img class="icon icon--sm" src="{% static 'images/icons/delete.svg' %}" alt="Delete"> </button> </div>

{% include '_partials/edit-delete-buttons.html' with edit_url=purchase_edit_url delete_url=purchase_delete_url %}

Step 1.2: Create _partials/table-header.html

{# templates/_partials/table-header.html #} {# Usage: {% include '_partials/table-header.html' with headers=header_list %} #} <div class="{{ grid_class|default:'grid' }} card-table-header"> {% for header in headers %} <div class="card-table-heading">{{ header }}</div> {% endfor %} </div>

Usage example:

{% include '_partials/table-header.html' with grid_class='grid-purchase' headers=purchase_headers %}

Step 1.3: Create _partials/action-button.html

{# templates/_partials/action-button.html #} {# For HTMX-enabled action buttons #} <button class="btn btn--{{ style|default:'primary' }}" hx-get="{{ url }}" hx-target="{{ target|default:'#modal-content' }}"> {{ icon|default:'+' }} {{ text }} </button>

{% include '_partials/action-button.html' with url=budgetitem_create_url text='Add Budget Item' icon='+' %}

  • ✅ Modularizes a monolithic 785-line CSS file into 13 focused modules
  • ✅ Extracts 166 lines of inline JavaScript into 6 separate, cacheable modules
  • ✅ Eliminates repetitive HTML patterns through Django template partials
  • ✅ Simplifies the grid system from 15+ specific classes to ~8 flexible utilities
  • ✅ Improves maintainability with clear file organization and BEM naming
  • ✅ Enhances accessibility with proper ARIA attributes and semantic HTML

The incremental, sprint-based approach ensures:

  • 🔒 Safety: Each sprint is fully tested before moving forward
  • 📈 Progressive improvement: Value delivered at each stage
  • 🔄 Flexibility: Sprints can be prioritized based on business needs
  • ⚡ Performance gains: Cached JavaScript and optimized CSS

Estimated Total Implementation Time: 38-52 hours
Can be completed in 1-2 weeks with dedicated development time
